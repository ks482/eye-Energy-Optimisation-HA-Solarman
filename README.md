
# DEO: Deye Energy Optimisation for Home Assistant

**DEO (Deye Energy Optimisation)** — це рішення для мінімізації витрат на електроенергію шляхом розумного керування зарядом акумуляторів в інверторах Deye (або їх аналогах) через Home Assistant за рахунок заряджання акумуляторів під час дії нічного тарифу до мінімально достатнього рівня до наступної дії нічного тарифу. Система аналізує прогноз сонячної генерації, історичні дані споживання та погодні умови, щоб забезпечити оптимальний рівень заряду. 


## 🚀 Основні можливості

-   **Динамічний нічний заряд:** Автоматичний розрахунок мінімально необхідного SOC (рівня заряду) на ніч (сутності time.deye12_program_6_time і time.deye12_program_1_time), щоб на основі даних денного споживання за останні 10 днів та з урахуванням прогнозу сонячної енергії на наступний день енергії вистачило до початку часу нічного тарифу наступного дня (до 23:00).
    
-   **Аналіз споживання:** Система робить "снепшоти" споживання вранці та ввечері для розрахунку середньодобового споживання електроенергії за останні 10 днів (лише за денний час).
    
-   **Telegram-звіти:** Отримання вечірнього звіту з прогнозом, дефіцитом та встановленими параметрами.

-   **Режим "Шторм":** Автоматичне перемикання на 100% заряд від мережі при виявленні сильного вітру чи опадів у прогнозі (або ручне увімкнення через дашборд). 
    
-   **Гнучке відновлення:** Після завершення негоди система повертає інвертор до ваших персональних налаштувань для кожної часової програми (сутності number.deye12_program_X_soc і select.deye12_program_X_charging).
  
-   **Налаштування через дашборд:**
    -  Ручне увімкнення режиму шторму (зарядити на 100% негайно і тримати заряд до вимкнення режиму) — input_boolean.deo_force_storm_mode
    -  Повна ємність вашої збірки АКБ у кВт⋅год (потрібно для переведення кВт⋅год дефіциту у відсотки SOC) — input_number.deo_battery_capacity
    -  Ваш «комфортний мінімум» на ніч. Нижче цього значення автоматика ніколи не опустить цільовий заряд, навіть якщо прогноз сонця ідеальний — input_number.deo_min_soc
    -  Нижня межа заряду акумулятора (наприклад, 15-20%) — input_number.deo_battery_reserve_soc
    -  Коефіцієнт недовіри до прогнозу (наприклад, 10% — це означає, що ми закладаємо, що сонця буде на 10% менше, ніж кажуть синоптики) — input_number.deo_forecast_buffer_percent
    -  Пороги вітру і дощу, при яких автоматично вмикається режим "Шторм" — input_number.deo_storm_wind_threshold / rain_threshold
    -  Налаштування input_boolean.deo_user_prog_2_chg_normal — input_boolean.deo_user_prog_5_chg_normal; input_number.deo_user_prog_2_soc_normal - input_number.deo_user_prog_6_soc_normal — режими зарядки та рівні SOC для денних програм, які система відновить після завершення режим "Шторм"

## 📋 Вимоги


Для роботи проєкту необхідно мати встановлені:

1.  **Home Assistant** (з налаштованим `configuration.yaml` або підтримкою `packages`).
    
2.  **Інтеграція ha-solarman (https://github.com/davidrapan/ha-solarman):** Сутності інвертора повинні мати стандартні назви (або бути адаптовані під ваші).
    
3.  **Інтеграція погоди Met.no (https://www.home-assistant.io/integrations/met):** Служба погоди, що надає щоденний прогноз.
    
4.  **Інтеграція Forecast Solar (https://www.home-assistant.io/integrations/forecast_solar):** Інтеграція для прогнозу генерації (сенсор `sensor.energy_production_tomorrow`).

5.  **Інтеграція Time & Date (https://www.home-assistant.io/integrations/time_date):** Для отримання часу початку і закінчення нічного тарифу (через сутності time.deye12_program_6_time (23:00) і time.deye12_program_2_time (07:00))
    
6.  **Інтеграція Telegram Bot (https://www.home-assistant.io/integrations/telegram_bot):** Для отримання сповіщень.
   
7.  **Mushroom Cards (https://github.com/piitaya/lovelace-mushroom):** Колекція карток для інтерфейсу панелі керування

*P.S. Ви можете використовувати інші інтеграції, змінивши відповідні сутності в коді.*

## 📂 Структура сутностей (Entities)

Усі сутності проєкту мають префікс `deo_` для зручного групування та фільтрації.

### Сенсори (Sensors)

| **ID** | **Опис** |
|--|--|
| `sensor.deo_required_soc_percent` | Фінальний розрахований SOC для нічного заряду |
|`sensor.deo_deficit_kwh` | Прогнозований дефіцит енергії в кВт⋅год|
| `sensor.deo_day_rate_mean_10d` | Середнє споживання будинку за останні 10 днів |
| `binary_sensor.deo_storm_mode` | Бінарний статус загрози (вітер/дощ) |


### 🛠️ Органи керування (Inputs)

| **ID** | **Опис** |
|--|--|
| `input_number.deo_battery_capacity` | Повна ємність вашої АКБ (кВт⋅год) |
| `input_number.deo_min_soc` | Залізний мінімум заряду на ніч |
| `input_number.deo_battery_reserve_soc` | Технічний резерв (Floor), нижче якого АКБ не розряджається |
| `input_boolean.deo_force_storm_mode` | Перемикач для примусової активації штормового режиму |


## 📦 Встановлення

1.  Створіть файл deye_energy_optimisation.yaml у папці `packages` (https://www.home-assistant.io/docs/configuration/packages) або скопіюйте вміст файлу безпосередньо в `configuration.yaml`  (перевірте, чи назви ваших сенсорів інвертора збігаються з використаними в коді (наприклад, `sensor.deye12_battery`)).
    
2.  Створіть ваш дашборд Lovelace для керування системою (наприклад, на основі файлу dashboard.yaml).
    
3.  Перезавантажте Home Assistant.
